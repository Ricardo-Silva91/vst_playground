No manual downloading. No zip extraction. No Gatekeeper headaches.

A single Python script that fetches the latest release info from GitHub, checks what's already installed on your system, and lets you install or update any plugin in one step. Runs on macOS and Windows. New plugins appear in the list automatically as they are released — no script updates needed.

macOS python3 install_lizard_vst.py
Windows python install_lizard_vst.py
Download install_lizard_vst.py
Requirements
Python 3

Comes pre-installed on macOS. On Windows, download from python.org if not already present — the installer checks during setup.

Internet connection

The script fetches release metadata from GitHub to check what's available and what has updates. The actual plugin bundles are downloaded only for what you choose to install.

Admin / sudo access

On macOS, the script asks for your password to write to /Library/Audio/Plug-Ins/VST3/. On Windows, a UAC prompt appears to allow writing to C:\Program Files\Common Files\VST3\.

How to Run
macOS

Open Terminal, navigate to the folder containing install_lizard_vst.py, and run:

python3 install_lizard_vst.py

The script will prompt for your password to install to the system VST3 folder and automatically clears the Gatekeeper quarantine flag — no "plugin is damaged" warnings.

Windows

Open Command Prompt, navigate to the folder containing install_lizard_vst.py, and run:

python install_lizard_vst.py

A UAC prompt will appear to allow writing to the system VST3 folder. Click Yes to proceed.

What It Does
Fetch release info› Check system VST3 folder› Show status table› Download & install› Clear Gatekeeper flag› Done

On launch the script fetches the latest release metadata from GitHub and compares it against what's installed in your system VST3 folder. It then shows a status table listing every available plugin with its release date and current status. You choose what to install — the script handles the rest.

Input Options
1, 2, 3 …

Install or update that specific plugin by number.

1,3

Install or update multiple plugins in one go.

a

Install or update everything available.

u

Install or update only plugins that have updates available.

q

Quit without installing anything.

After Installing
▶

Rescan plugins in your DAW so it picks up new or updated plugins. In FL Studio: Options → Manage plugins → Find more plugins. Other DAWs: use the VST3 scan or plugin manager in settings.

▶

New plugins appear automatically. Any plugin added to the collection that has been built at least once will appear in the status table next time you run the script — no need to update the installer itself.

Troubleshooting
macOS — "plugin is damaged" after installing

The script runs xattr -cr automatically. If the warning still appears after a fresh install, run the script again — it will overwrite the bundle and clear the quarantine flag.

Windows — plugin not appearing after install

Make sure your DAW is scanning C:\Program Files\Common Files\VST3\. In FL Studio: Options → Manage plugins → add that path under VST3 scan locations if it's missing, then rescan.

"No plugins found"

Check your internet connection. If GitHub is reachable but the list is empty, a build may not have run yet for any plugin — this should resolve itself once the next build completes.
